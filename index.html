Demo API

API xác minh vị trí thế hệ mới dựa trên dữ liệu mạng di động

API cho phép xác minh thiết bị di động có đang hiện diện trong một khu vực địa lý cụ thể hay không. Thay vì trả về vị trí chi tiết, API phản hồi theo logic xác minh TRUE, FALSE hoặc PARTIAL để bảo vệ quyền riêng tư trong khi vẫn đáp ứng nhu cầu nghiệp vụ.

Ví dụ request xác minh thiết bị trong vùng hình tròn với tâm tọa độ, bán kính và maxAge.

Xem request/response mẫu
POST /location-verification/v1/verify 
Authorization: Bearer <access_token>
Content-Type: application/json

{ 
  "device": { "phoneNumber": "+123456789" },
  "area": {
    "areaType": "CIRCLE",
    "center": {
      "latitude": 10.7769,
      "longitude": 106.7009
    },
    "radius": 500
  },
  "maxAge": 600
}

HTTP 200
{
  "verificationResult": "TRUE",
  "lastLocationTime": "2026-05-14T09:15:00Z"
}
Dữ liệu đầu vào

Tối giản. Đủ để xác minh. Không lấy thừa.

TrườngMô tả
phoneNumber / network IDĐịnh danh thiết bị cần xác minh.
areaType = CIRCLEKiểu vùng xác minh hiện hỗ trợ hình tròn.
latitude, longitudeTâm của vùng cần xác minh.
radiusBán kính vùng xác minh.
maxAgeTuổi tối đa chấp nhận được của dữ liệu vị trí.
Kết quả trả về

Ba trạng thái chính, xử lý linh hoạt theo nghiệp vụ

TRUE

Thiết bị nằm hoàn toàn trong vùng yêu cầu.

FALSE

Thiết bị không nằm trong vùng yêu cầu.

PARTIAL

Thiết bị khớp một phần, kèm matchRate để ước lượng xác suất.

Trạng tháiÝ nghĩa
UNABLE_TO_LOCATEMạng không thể định vị thiết bị tại thời điểm yêu cầu.
AREA_NOT_COVEREDVùng yêu cầu nằm ngoài phạm vi hỗ trợ hoặc vi phạm giới hạn pháp lý.
UNABLE_TO_FULFILL_MAX_AGEDữ liệu vị trí hiện có cũ hơn ngưỡng maxAge cho phép.
Cách thức hoạt động

Đơn giản. Nhanh. Dựa trên dữ liệu mạng.

1

Gửi request

Ứng dụng/đối tác gửi request qua gateway của nhà mạng hoặc aggregator Open Gateway.

2

Xác định vùng cần kiểm tra

Request gồm định danh thiết bị và khu vực cần xác minh, là hình tròn với tâm tọa độ và bán kính.

3

Đối chiếu mạng di động

Nhà mạng đối chiếu thông tin định vị từ mạng di động, tuân thủ điều kiện consent và phạm vi coverage.

4

Nhận kết quả

API trả về thiết bị có trong vùng, không trong vùng, hay chỉ khớp một phần.

5

Ra quyết định nghiệp vụ

Ứng dụng dùng kết quả để cho phép, từ chối, yêu cầu xác minh bổ sung hoặc ghi nhận rủi ro.

Bảo mật & quyền riêng tư

Xây dựng trên nền tảng consent và tuân thủ

CAMARA Device Location Verification API cần được triển khai theo nguyên tắc bảo mật, minh bạch và tôn trọng quyền riêng tư. API hỗ trợ các luồng xác thực OpenID Connect và chỉ nên sử dụng kết quả xác minh ở mức cần thiết cho mục đích nghiệp vụ.

  • ✓Hỗ trợ các luồng xác thực OpenID Connect: CIBA và Authorization Code Flow theo chuẩn Open Gateway.
  • ✓Trong mô hình 3-legged token, danh tính người dùng được xác định từ token, không cần truyền lại thông tin thiết bị.
  • ✓Chỉ dùng kết quả xác minh ở mức cần thiết, không lưu trữ dữ liệu vị trí thô nếu không có căn cứ pháp lý.
  • ✓Cần thiết kế consent UX rõ ràng: ai yêu cầu, mục đích gì, thời hạn bao lâu, người dùng có quyền kiểm soát thế nào.
Lưu ý khi tích hợp

Một số điểm cần lưu ý trước khi triển khai

01Kiểm tra phiên bản API: v1/v2/v3 có thể khác nhau về trường dữ liệu và mã lỗi.
02Không ra quyết định chỉ từ một tín hiệu: kết hợp SIM Swap, Number Verification, device risk, fraud scoring.
03Cân chỉnh radius và maxAge phù hợp nghiệp vụ: radius quá nhỏ tăng false negative, maxAge quá ngắn tăng lỗi.
04Chuẩn hóa xử lý lỗi: unable to locate, area not covered, timeout, consent denied.
05Kiểm thử theo nhà mạng và vùng phủ sóng vì độ chính xác phụ thuộc điều kiện hạ tầng.
06Thiết kế audit log đầy đủ: ai gọi, mục đích, kết quả, thời điểm; không lưu trữ dữ liệu vị trí vượt mức cần thiết.

FAQ

Câu hỏi thường gặp

API có trả về tọa độ chính xác của người dùng không?

Không. API xác minh thiết bị có hiện diện trong một khu vực địa lý cụ thể hay không và phản hồi theo logic TRUE, FALSE hoặc PARTIAL.

API dùng dữ liệu nào để xác minh?

API dựa trên thông tin định vị từ mạng di động, thông qua gateway của nhà mạng hoặc aggregator Open Gateway.

Khu vực xác minh được mô tả như thế nào?

Request gồm khu vực hình tròn với tâm tọa độ và bán kính.

PARTIAL có nghĩa là gì?

Thiết bị khớp một phần, kèm matchRate để ước lượng xác suất.

Cần lưu ý gì về consent?

Cần thiết kế consent UX rõ ràng: ai yêu cầu, mục đích gì, thời hạn bao lâu và người dùng có quyền kiểm soát thế nào.

Có nên dùng API này làm tín hiệu quyết định duy nhất không?

Không. Nên kết hợp với SIM Swap, Number Verification, device risk và fraud scoring.
